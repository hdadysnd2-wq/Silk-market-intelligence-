**ملاحظة صدق:** عيّنات `/analyze` الأربع أعلاه (باستثناء `research_report_
latest.docx`) وُلّدت معاً من تشغيلة `silk_engine.analyze` حتمية واحدة
(محرّك حقيقي + مخزن حقائق مبذور بسوقين: الصين والإمارات) داخل حاجز شبكة
صادق — لا اختلاق؛ فشل الجلب الحي (Comtrade/World Bank/FAOSTAT) مُعلَن
كفجوة في التقرير نفسه لا مُخفى. أعد توليدها بعد أي تعديل على طبقة العرض
(قاعدة §10.6):

```bash
python3 tools/gen_analyze_samples.py
```

لتوليد نموذج ببيانات حية بدل المبذورة، شغّل نفس السكربت من بيئة متصلة
بالشبكة (النشر) — الحاجز الحتمي داخل السكربت يبقى، فالنتيجة ستقرأ من
المخزن أولاً كما في الإنتاج؛ لتحليل حي كامل استخدم `silk_engine.analyze`
مباشرة بلا حاجز الشبكة.
